Prototype workflow

How the SODI Material Library works

This prototype is designed for a GitHub Pages site. Bookings are stored in the browser for demonstration, so it can be shown in class without a database or login system.

01

Search

Students search for a material by name, studio use, condition, or storage bay.

02

Check stock

Each card shows the available amount, booking unit, condition, and suggested project uses.

03

Reserve

Students choose an amount and submit a simple reservation for collection from the Te Aro material shelf.

04

Return or donate

Useful offcuts can come back into the library so the material loop stays active.

Material intake

Donate leftover project material

Use this section as a prototype intake point for students who have clean, reusable material after a project hand-in. It does not send data anywhere yet, but it shows how the next version could collect donations.
